# 1150625 豪雨災點共同填報系統 — 部署說明

架構:**GitHub Pages(前端地圖網頁,免費靜態託管)+ Google 試算表 + Apps Script(資料庫,免費)**。
所有人打開網址即可在地圖上填報每個災點為「舊災點 / 新災點 / 非積淹水事件」,結果即時寫入 Google 試算表並同步給所有使用者,最後可一鍵匯出與「總表」欄位完全相同的 CSV。

---

## 步驟一:建立資料庫(Google 試算表 + Apps Script,約 5 分鐘)

1. 到 https://sheets.google.com 建立一個新的空白試算表,命名例如「1150625災點填報資料庫」。
2. 上方選單 → **擴充功能 → Apps Script**。
3. 刪除編輯器內預設程式碼,把隨附的 **`Code.gs`** 全部內容貼上,按儲存(Ctrl+S)。
4. 右上角 **部署 → 新增部署作業**:
   - 類型齒輪 → 選 **網頁應用程式**
   - 「執行身分」→ **我**
   - 「誰可以存取」→ **所有人**(這樣同仁不需登入 Google 就能填報)
   - 按「部署」,首次會要求授權 → 選你的帳號 → 進階 → 前往(不安全)→ 允許
5. 複製產生的**網頁應用程式網址**,格式為
   `https://script.google.com/macros/s/XXXXXXXX/exec` ← 這就是資料庫網址(API_URL)。

> 之後若修改 Code.gs,要用「部署 → 管理部署作業 → 編輯 → 版本:新版本」更新,網址才會生效。

## 步驟二:設定網頁的資料庫網址

1. 用記事本/VS Code 打開 `1150625豪雨災點地圖_共同填報版.html`。
2. 搜尋 `const API_URL`(在檔案的 script 開頭,有醒目註解框),把網址貼進引號:
   ```js
   const API_URL = "https://script.google.com/macros/s/XXXXXXXX/exec";
   ```
3. 存檔。
   (測試時也可以先不改檔案,在網頁左側面板按「設定資料庫網址」臨時貼上,但該設定僅當次有效。)

## 步驟三:發佈到 GitHub Pages

1. 登入 GitHub → 右上 **+ → New repository** → 名稱例如 `flood-0625`,設 Public → Create。
2. **Add file → Upload files**,上傳步驟二改好的 HTML 檔,並**改名為 `index.html`**(在上傳畫面點檔名即可改)→ Commit。
3. 儲存庫 **Settings → Pages** → Source 選 `Deploy from a branch`,Branch 選 `main` / `(root)` → Save。
4. 約 1 分鐘後網址生效:`https://你的帳號.github.io/flood-0625/`
   把這個網址發給大家即可開始填報。GitHub Pages 是 HTTPS,手機 GPS 定位功能可正常使用。

## 日常使用流程

- **填報**:點地圖上任一災點 → 彈窗按「📝 填報判定」→ 三選一(舊災點會列出同區 108-115 災點下拉選單,系統名稱比對相符者以★置頂並預選)→ 填上單位/姓名 → 送出。已填報的點會出現**綠色外環**與「✓已填報」徽章。
- **同步**:每次開啟網頁自動載入最新填報;填報過程中可按「🔄 重新整理填報資料」取得他人最新結果。同一序號重複填報以**最新一筆為準**,試算表內保留全部歷程供稽核。
- **未定位災點**:手動標點或 GPS 現地定位後再填報,座標會一併寫入資料庫,其他人開啟時自動套用(定位也連動)。
- **產出總表**:按「匯出總表格式CSV」→ 得到與原「總表(依行政區排序)」欄位、順序完全相同的檔案(序號/轄區科/行政區/淹水路段(地址)/淹水高度/最大水深/最大水深時間/非積淹水事件原因/新災點合併災點名稱/舊災點-108-115積淹水點/權責單位/淹水時間/退水時間/通報來源/備註/座標X/座標Y),用 Excel 開啟後貼回原總表即可。

## 注意事項

- 本系統範圍為 7 個轄區(仁德、東、中西、北、安南、安平、南區)共 364 筆;其他行政區如需納入,再提供需求即可擴充。
- 「誰可以存取:所有人」代表知道網址者皆可填報(填報人欄位靠自律填寫)。若需限制,可改為「僅限機關網域」,但填報者需登入該網域 Google 帳號。
- Google 圖磚底圖供內部作業使用;正式對外公開建議切換為通用電子地圖/OSM/魯地圖底圖為主。
- Apps Script 免費額度(每日數萬次讀寫)遠超過本案需求,無須擔心。
